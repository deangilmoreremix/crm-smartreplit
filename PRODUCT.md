# Product

## Register

product

## Users

**Primary users:**
- Sales teams & CRM power users who manage contacts, deals, pipelines, and communications daily
- Agency owners & white-label partners who resell/rebrand the CRM for their clients

**Context:**
- Sales teams work in fast-paced environments, need quick access to customer data, deal intelligence, and communication tools
- Agency owners manage multiple client tenants, configure branding, and monitor usage
- Both user types expect enterprise-grade reliability with consumer-grade ease of use

**Jobs to be done:**
- Track and manage customer relationships across the full lifecycle
- Close deals faster with AI-powered insights and automation
- Manage multi-tenant deployments with custom branding
- Access 60+ AI tools for content, analytics, and outreach

## Product Purpose

SmartCRM is an AI-powered customer relationship management platform that combines traditional CRM functionality with cutting-edge AI capabilities, multi-tenant architecture, and comprehensive business automation tools. Built on the twentyCRM foundation with significant enhancements, the platform serves both direct users and white-label partners.

**Success looks like:**
- 35% improvement in sales velocity
- 50% faster contact data enrichment
- 40% improvement in goal achievement rates
- 85% feature utilization rate
- 4.8/5.0 user satisfaction score

## Brand Personality

**Bold, innovative, energetic**

- **Bold**: Confident design choices that stand out from generic SaaS. Not afraid of color, motion, or distinctive typography when appropriate.
- **Innovative**: AI-first positioning reflected in the interface — smart suggestions, predictive analytics, natural language interfaces.
- **Energetic**: Dynamic, alive interfaces with purposeful motion and clear feedback. Not static or stale.

**Emotional goals:**
- Users should feel empowered and in control
- The AI should feel like a capable assistant, not a replacement
- The interface should convey competence and modernity

## Anti-references

- **Generic SaaS templates**: Inter font everywhere, purple-to-blue gradients, cards nested in cards, gray text on colored backgrounds, rounded-square icon tiles above every heading
- **Overly corporate/conservative design**: Dense tables, boring blues, no personality
- **Cluttered/overwhelming interfaces**: Too many features visible at once without progressive disclosure
- **Minimalism that hides functionality**: Too clean, important features buried behind too many clicks
- **AI slop aesthetics**: Hero metric templates (big number + small label + gradient), identical card grids, glassmorphism as default, gradient text, side-stripe borders

## Design Principles

1. **AI is the co-pilot, not the pilot** — AI features should enhance human decision-making, not replace it. The interface should make AI capabilities discoverable without being intrusive.

2. **Power through clarity** — Complex CRM data should be presented clearly. Dense information is acceptable when well-organized; clutter is not.

3. **Consistency across tenants** — White-label partners can customize branding, but core UX patterns remain consistent so users can navigate any tenant instance intuitively.

4. **Progressive disclosure for complexity** — With 60+ AI features and deep CRM functionality, reveal complexity only when needed. Default views should show the 20% of features used 80% of the time.

5. **Motion with purpose** — Animations and transitions should convey state changes and guide attention, not decorate. Every motion should answer a user question ("What just happened?" "Where did that go?").

## Accessibility & Inclusion

- **WCAG AAA** compliance target where possible
- **Reduced motion support**: All animations must respect `prefers-reduced-motion`
- **Color blindness**: Critical information must not rely on color alone; use patterns, labels, and icons
- **Keyboard navigation**: Full keyboard accessibility for all interactive elements
- **Screen reader support**: Proper ARIA labels, landmarks, and live regions for dynamic content
- **Touch targets**: Minimum 44x44px for all interactive elements on touch devices
- **Text scaling**: Layouts must not break at 200% zoom
